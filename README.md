# College HelpDesk

`College HelpDesk` — учебное веб-приложение по дисциплине «Тестирование информационных систем». Это локальный сервис заявок колледжа: пользователи сообщают о проблемах с техникой, доступами, аудиториями, `1С` и другими учебными сервисами, а сотрудники обрабатывают обращения.

Проект сделан на простом стеке:

- Python 3
- Flask
- SQLite
- HTML
- CSS
- JavaScript

Приложение запускается локально без Docker, внешних API и отдельной настройки базы данных.

## Установка

1. Откройте терминал в папке проекта.
2. Установите зависимости:

   ```bash
   pip install -r requirements.txt
   ```

3. Запустите сервер:

   ```bash
   python app.py
   ```

4. Откройте сайт:

   `http://127.0.0.1:5000`

При первом запуске рядом с кодом автоматически создаётся база `college_helpdesk.db` с тестовыми данными.

Важно: приложение запускается через Flask. Файл `index.html` в корне — только памятка для локального запуска, а не основной экран сервиса.

## Тестовые аккаунты

| Роль | Логин | Пароль |
| --- | --- | --- |
| Студент | `student1` | `student123` |
| Преподаватель | `teacher1` | `teacher123` |
| Исполнитель | `executor1` | `executor123` |
| Администратор | `admin1` | `admin123` |

## Роли

- `student` создаёт обращения и отслеживает свои заявки.
- `teacher` создаёт обращения по учебным аудиториям и сервисам.
- `executor` обрабатывает назначенные заявки и меняет их статусы.
- `admin` контролирует очередь, назначает исполнителей и просматривает пользователей.

## Функции сервиса

- вход по логину и паролю;
- главная страница со статистикой;
- список заявок с фильтрами, поиском и сортировкой;
- создание заявки с темой, категорией, аудиторией, приоритетом, описанием и контактом;
- карточка заявки с автором, исполнителем, комментариями и историей статусов;
- добавление комментариев;
- изменение статуса заявки;
- админ-панель со статистикой;
- страница пользователей для административной работы.

В тестовой базе есть заявки с разными категориями, статусами и приоритетами, комментарии и несколько пользователей.

## Что тестировать

Во время проверки обращайте внимание на:

- авторизацию и доступность страниц для разных ролей;
- создание и отображение заявок;
- поля форм и обработку некорректных данных;
- список заявок, фильтры, поиск и сортировку;
- комментарии, статусы, исполнителей и историю изменений;
- статистику на главной странице и в админ-панели;
- работу интерфейса на разных размерах экрана;
- сохранность данных после действий пользователя.

## Задание для студентов

Студентам нужно изучить сервис, описать его функции, составить чек-лист, подготовить тест-кейсы, провести тестирование, оформить баг-репорты, предложить приоритет исправлений, выполнить ретест после исправлений и подготовить итоговый отчёт о качестве системы.
